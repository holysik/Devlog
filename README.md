## 📌 프로젝트 개요 (Project Overview)

최근 개발자 커뮤니티와 기술 공유 플랫폼이 증가하면서  
다양한 서비스를 직접 사용해보며 구조를 분석했습니다.

기존 서비스들은 기술 기록 중심으로 구성되어 있어  
사용자 간 상호작용이 부족하고,  
콘텐츠 생산에 대한 보상이 연결되지 않는 한계가 있다고 느꼈습니다.

이를 개선하기 위해 본 프로젝트는 기술 공유를 넘어,  
사용자 간 상호작용이 가능한 SNS 구조와  
양질의 콘텐츠가 수익으로 이어지는 구조를 함께 고려한  
개발자 커뮤니티 플랫폼으로 기획했습니다.

이를 통해 참여와 보상이 연결되는 지속 가능한 커뮤니티를 구현하고자 했습니다.

## 🧪 문제 해결

### 💥 Case: 채팅방 내 메시지별 읽은 회원 수 처리 문제

#### 🧩 Problem

채팅방에서 메시지별 읽은 회원 수를 처리하는 과정에서,
읽음 처리를 입장 이벤트 기준으로 감소시키는 방식으로 구현하다 보니
다수 사용자가 동시에 입장·퇴장하는 상황에서 실제 읽은 회원 수와 불일치하는 문제가 발생

#### 🔎 Cause
 - 읽음 카운트를 사용자의 실제 읽음 상태가 아닌 입장 이벤트에 의존하여 감소 처리
 - 동일 사용자가 반복 입장할 경우 카운트가 중복 감소하는 문제 발생
   
#### 🛠 Solution

읽음 처리 방식을 실시간 처리와 기준 계산으로 분리하여 개선

 - 실시간 처리 (메모리 기반)
   - 현재 채팅 중인 사용자 정보를 ConcurrentHashMap으로 관리
   - 실시간으로 읽음 상태를 반영하여 UI에 즉시 반영
 - 기준 계산 (DB 기반)
  - 사용자별 lastReadMessage를 기준으로 읽지 않은 메시지를 계산
  - 채팅방 입장 시 해당 기준을 기반으로 읽음 상태 재계산

#### ✅ Result
 - 동시 접속 및 입·퇴장 상황에서도 메시지별 읽은 회원 수 정확도 확보
 - 동일 사용자 반복 입장으로 인한 카운트 오류 제거
 - 읽음 처리 로직 단순화 및 유지보수성 향상
